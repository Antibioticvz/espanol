# Руководство пользователя: Combine + Audio Learner

**Версия:** 1.2  
**Дата:** 21 июля 2026

Это практическое руководство для преподавателя (вы будете и создавать уроки в Combine, и учить фразы в Audio Learner на iPhone).

**Что нового в 1.1 → 1.2** (если вы уже знакомы с версией 1.0, вот что изменилось):
- **Режим флеш-карт** — интерактивные карточки со свайпами вместо аудио-очереди (§6.3)
- **Сессия дня** — кросс-урочный подбор фраз к повтору сразу по всем урокам (§6.1)
- **Пропорциональные паузы, порядок сторон (в т.ч. shadowing), автоскорость** — новые настройки аудио-сессии (§6.2)
- **Sleep-таймер** плеера (§6.2)
- **Кнопка на Home Screen виджете, Siri, Live Activity/Dynamic Island** — запуск и отслеживание сессии без открытия приложения (§6.5)
- **Нормализация громкости** фраз при генерации (§1.5)
- **Персистентный API-ключ** ElevenLabs в Combine (§1.4)
- **Экспорт в Anki** (.apkg) и **пакетная генерация из CLI** (§4.3, §3.6)
- DMG-сборка и установка Combine (§1.1)

---

## Быстрый старт

### Что вам потребуется
- **macOS** с установленным Combine (скачайте `.dmg`)
- **iPhone** с установленным Audio Learner
- **Текстовый файл** с уроком (в простом формате разметки)
- **ElevenLabs API-ключ** (опционально; без него Combine работает бесплатно в режиме `mock_say`)

## Часть 1: Combine на Desktop

### 1.1 Первый запуск Combine

1. **Скачайте и установите** Combine (файл вида `Combine-1.1.0.dmg` — точная версия в имени
   зависит от того, когда собран установщик; актуальную команду сборки и версию смотрите в
   `combine/README.md`)
   - Откройте DMG
   - Перетащите Combine.app в папку Applications
   - Запустите Combine

   Сборка **не подписана** сертификатом Apple Developer (личное использование без публикации) —
   при первом запуске macOS Gatekeeper, скорее всего, покажет «Combine нельзя открыть, так как
   не удаётся проверить разработчика». Откройте через **правый клик по Combine.app → Открыть**
   (подтвердите во всплывающем окне), либо через Системные настройки → Конфиденциальность и
   безопасность → «Всё равно открыть».

2. Отдельного экрана-приветствия с настройкой при первом запуске нет — Combine сразу открывается
   на экране **«Импорт текста»** с уже применёнными разумными настройками по умолчанию:
   - Папка для сохранения уроков: `~/lessons` (меняется в любой момент в [Параметры])
   - Провайдер: `mock_say` (бесплатный режим на голосах macOS) — ElevenLabs API-ключ можно
     добавить позже, когда он понадобится, в [Параметры]

### 1.2 Два режима работы

Провайдер выбирается на экране настроек, в выпадающем списке «Провайдер»:

#### Режим 1: Mock (бесплатный)

`Mock (macOS say — бесплатно)` — Combine генерирует уроки через встроенный голос macOS (`say`). Достаточно для проверки структуры и UX, но качество зависит от голосов, установленных в системе.

**Преимущества:**
- Бесплатно, неограниченно
- Полнофункциональна работа Combine (включая нормализацию громкости, ID3-теги и т.д.)
- Хорошо для разработки и тестирования

#### Режим 2: ElevenLabs (платный, лучшее качество)

`ElevenLabs (реальный API)` — Combine использует реальный API ElevenLabs для синтеза профессионального качества.

**Как получить ключ:**
1. Перейдите на https://elevenlabs.io
2. Создайте аккаунт (Free или Pay-as-you-go)
3. Перейдите в Settings → API Keys
4. Скопируйте ваш ключ (начинается с `sk-`)

**Как включить:**
- На экране настроек выберите провайдер `ElevenLabs (реальный API)`
- Появится поле «API-ключ ElevenLabs» — вставьте ваш ключ и нажмите **«Сохранить»** (ключ
  сохраняется зашифрованным на диске — см. §1.4)
- Нажмите **«Проверить подключение»**, чтобы убедиться, что ключ рабочий (✓/✗ с сообщением)

**Стоимость:**
- ~$0.10 за 1000 символов (Multilingual v2)
- ~$0.05 за 1000 символов (Flash v2.5)
- Курс из нескольких десятков тем — от нескольких десятков долларов, зависит от объёма текста
  (точную оценку для вашего урока Combine считает на экране настроек — см. §3.1)

### 1.3 Тестовая генерация (очень важно!)

**Зачем:** проверить, что ключ работает, выбрать подходящий голос, оценить стоимость — всё на короткой фразе (несколько центов с реальным ElevenLabs-ключом, бесплатно в mock-режиме).

**Как сделать:**

1. На экране настроек найдите секцию **«Тестовая генерация»**
2. Поле по умолчанию: `Hola, ¿cómo estás?` — можно заменить на любой другой текст (до 200 символов)
3. Выберите голос в выпадающем списке «Голос» (что в нём — см. §3.1: список зависит от провайдера)
4. Нажмите **«▶ Сгенерировать тест»**
5. Через 2–5 сек прямо под кнопкой появится аудиоплеер и строка с результатом
6. Если качество нравится → можно генерировать весь урок

**Результат** (показывается в той же панели, под кнопкой, а не отдельным баннером вверху экрана):
- ✓ Ключ и голос работают, если ошибки нет (иначе — текст ошибки красным под кнопкой)
- ✓ Аудио сразу проигрывается во встроенном плеере
- ✓ Строка с фактическими цифрами: «Длительность: 1.8 сек · Символов: 19 · Стоимость: $0.002»

### 1.4 Персистентность API-ключа ElevenLabs (новое в 1.2)

Раньше ключ жил только в памяти текущего запуска Combine — при каждом перезапуске его нужно
было вводить заново. Теперь ключ можно сохранить один раз:

1. Введите ключ в поле «API-ключ ElevenLabs» и нажмите **«Сохранить»**
2. Ключ шифруется через `safeStorage` Electron (на macOS этот механизм опирается на Keychain) и
   хранится на диске только в зашифрованном виде; сам ключ никогда не возвращается обратно в
   интерфейс — оттуда приходит только статус («ключ сохранён» / не задан)
3. При следующем запуске поле показывает `•••••••• (ключ сохранён)` вместо пустого — вводить
   заново не нужно: генерация и тестовая генерация используют сохранённый ключ автоматически
4. **«Изменить»** — ввести другой ключ вместо сохранённого. **«Удалить ключ»** — стереть его
   (например, если ключ был отозван на elevenlabs.io)

Если шифрованное хранилище недоступно на вашей сборке/платформе, Combine честно предупредит об
этом («ключ будет использован только в рамках текущего запуска») и продолжит работать как
раньше, без сохранения на диск. Если сохранённый ключ повреждён (например, файл скопирован с
другого Mac, где использовался другой ключ шифрования) — Combine покажет «Сохранённый ключ
повреждён» и попросит ввести его заново.

### 1.5 Нормализация громкости фраз (новое в 1.2)

Типичная проблема TTS-контента: разные фразы одного урока могут звучать заметно громче или тише
друг друга. Переключатель **«Нормализация громкости фраз»** в параметрах генерации (включён по
умолчанию) выравнивает уровень:

- **mock_say** — нормализуется всегда силами самого Combine (чистый JS, целевой уровень ≈ −20
  dBFS RMS с защитой от клиппинга) — ffmpeg для этого не требуется.
- **ElevenLabs** — требует **ffmpeg** в `PATH` (готовый MP3 прогоняется через `loudnorm`, EBU
  R128, −18 LUFS). Если ffmpeg не установлен, Combine покажет прямо под переключателем
  предупреждение:

  > Нормализация недоступна: установите ffmpeg (напр. `brew install ffmpeg`) — без него фразы
  > ElevenLabs сохраняются без изменения громкости. mock_say нормализуется всегда (не требует
  > ffmpeg).

  Установите ffmpeg (`brew install ffmpeg`) и запустите генерацию заново, либо продолжайте без
  него — фразы всё равно сгенерируются, просто без выравнивания громкости.

## Часть 2: Подготовка урока

### 2.1 Формат текста

Создайте текстовый файл (`.txt` или `.md`) вот в таком формате:

```
#TOPIC 4 | Рассказ о себе

##BLOCK verb_group | Кто я — происхождение и факты

#WORD llamarse | зваться
Me llamo Victor. | Меня зовут Виктор.
¿Cómo te llamas tú? | Как тебя зовут?

##BLOCK phrase_group | Ходовые фразы

#CATEGORY Первое знакомство
Mucho gusto, soy Victor. | Очень приятно, я Виктор.

##BLOCK vocabulary | Ключевая лексика
el programador | программист
la familia | семья

##BLOCK story | Короткий рассказ
ES: Me llamo Victor...
RU: Меня зовут Виктор...
```

**Основные правила:**
- ✓ Разделитель между текстом и переводом: `|` (пробелы вокруг него — только для читаемости,
  не обязательны)
- ✓ Только один `|` на строку фразы
- ✓ Пустые строки между блоками (они автоматически пропускаются)
- ✓ Чувствителен к формату #TOPIC и ##BLOCK
- ✓ Блок `story` — ровно две строки: `ES:` и сразу за ней `RU:`, каждая одной строкой без переноса

**Подробнее:** см. документ `LESSON_FORMAT.md`

### 2.2 Загрузка в Combine

1. **Экран 1: Импорт текста**
   - Нажмите **[📄 Загрузить файл]** (выберите `.txt`/`.md`)
   - Или вставьте текст прямо в большое текстовое поле ниже — оно доступно всегда, отдельной
     кнопки «Вставить вручную» нет. Рядом есть кнопка **[Вставить пример]** — она подставляет
     готовый образец урока (удобно, чтобы посмотреть формат, а не для вставки своего текста)
   - Combine парсит в реальном времени, показывает:
     - ✓ «Тема найдена: "Рассказ о себе"»
     - ✓ «Блоков: 4, Фраз: 81, Ошибок: 0»

2. **Проверьте валидацию:**
   - ✗ Красным: ошибки парсинга (номер строки + описание)
   - ⚠️ Жёлтым: предупреждения (например, пустая категория)
   - ✓ Зелёным: готово к генерации

3. Нажмите **[Далее →]** если всё окей

## Часть 3: Генерация урока

### 3.1 Экран настроек

**Левая панель — ElevenLabs параметры:**

| Параметр | Значение | Совет |
|----------|---------|-------|
| **API Key** | sk-••••• | Сохраняется зашифрованным на диске — вводить каждый раз не нужно (см. §1.4) |
| **Модель** | Multilingual v2 или Flash v2.5 (или указать любой ID модели вручную) | Flash дешевле в 2 раза, но менее полирован |
| **Голос испанский / Голос русский** | Список подтягивается динамически из вашего аккаунта ElevenLabs (`GET /v1/voices`) — конкретных имён вроде «Pablo»/«Masha» в реальном UI нет, это были вымышленные плейсхолдеры старой спеки | Прослушайте вариант кнопкой **«🔊 Слушать»** перед тем как зафиксировать выбор — поменять голос для уже сгенерированного урока нельзя, только переделать генерацию заново |
| **Stability** | 0.0–1.0 (по умолчанию 0.5) | Высокая = монотонная, низкая = вариативная |
| **Similarity Boost** | 0.0–1.0 (по умолчанию 0.75) | Насколько точно копировать голос |

В mock-режиме (`mock_say`) вместо аккаунта ElevenLabs список голосов — это голоса, установленные
в macOS (по умолчанию `Mónica` для испанского и `Milena` для русского; если их нет на машине —
Combine сам подбирает ближайший установленный голос нужного языка, а если и такого нет —
системный голос по умолчанию, с предупреждением в логе).

**Правая панель — параметры генерации:**

| Параметр | Значение | Совет |
|----------|---------|-------|
| **Папка вывода** | ~/lessons | Куда сохранять готовые уроки |
| **Параллельные запросы** | 1–5 (по умолчанию 3) | Больше = быстрее, но риск ошибок (429) |
| **Макс попыток** | 1–10 (по умолчанию 3) | Сколько раз повторять при ошибках |
| **Delay** | 100 ms | Пауза между запросами (выше = медленнее, но безопаснее) |
| **Timeout** | 30 сек | Максимум ждать ответа API |
| **Нормализация громкости фраз** | вкл/выкл (по умолчанию вкл) | См. §1.5 — для ElevenLabs нужен ffmpeg |

**Калькулятор стоимости** (внизу):
```
Модель: Multilingual v2 ($0.10 / 1000 сим)
Языков: 2
Фраз всего: 81

Символов:
  ES: 3 737
  RU: 3 799
  ИТОГО: 7 536

Цена: 7 536 × $0.10 / 1000 = $0.75
```

⚠️ Это оценка (±5%). Реальная цена может отличаться.

### 3.2 Запуск генерации

1. **Убедитесь:** API-ключ работает (кнопка «Проверить подключение» должна показать ✓)
2. **Нажмите [Генерировать →]**

### 3.3 Экран генерации

Вы увидите прогресс в реальном времени:

```
Генерация: Рассказ о себе

Статус: [████████════] 66% (54 / 81 фраз)

Текущий элемент:
💬 04-b1-llamarse-02 (ES)
   "¿Cómo te llamas tú?"
   [████████───] 3.2 / 4.8 сек

Статистика:
Скорость: 12.4 фраз/мин
Осталось: ~3 мин
Успешно: 54 | Ошибок: 0 | В очереди: 27
Потрачено: $0.41 / $0.75 (55%)
```

**Дерево блоков (разворачиваемое):**
- ▼ BLOCK 1: verb_group (20 фраз) [████████──]
  - ▼ llamarse (5 фраз) [██████████]
    - ✓ 04-b1-llamarse-01 ES (1.2 сек)
    - ✓ 04-b1-llamarse-01 RU (1.1 сек)
    - ⏱ 04-b1-llamarse-02 ES [generating...]
  - ▼ tener (6 фраз) [██████░░░░]

**Кнопки управления:**
- ⏸ **Пауза** — остановить, возобновить позже
- ⏹ **Отмена** — остановить без сохранения (потерять незаконченное)

### 3.4 При ошибке 429 (лимит)

Combine автоматически:
1. Обнаруживает 429 Too Many Requests
2. Ждёт (exponential backoff: 1s → 2s → 4s)
3. Повторяет (макс попыток — по настройке, по умолчанию 3)
4. Если не помогло → помечает фразу как `failed`

**Что делать:**
- ✓ Подождите (лимит восстановится за ~1 час)
- ✓ В Библиотеке нажмите [▶ Возобновить] на карточке урока → Combine продолжит с `failed` фраз

### 3.5 Завершение

Когда все фразы сгенерированы:

```
✓ Урок "Рассказ о себе" готов!

Папка: ~/lessons/04-hablar-de-mi-mismo/
Статистика:
  • 81 фраза + 15 слов + 1 рассказ
  • 162 аудиофайла
  • 34.2 МБ
  • Затрачено: $0.74

[🎵 Проиграть] [📂 Открыть в Finder] [Далее →]
```

Нажмите **[📂 Открыть в Finder]** — папка `04-hablar-de-mi-mismo/` будет выделена.

### 3.6 Пакетная генерация нескольких тем (CLI, новое в 1.1)

Кнопки «Сгенерировать все недостающие темы» в самом интерфейсе Combine нет — для пакетной
обработки используется терминал:

```bash
cd combine
npm run cli -- generate --input ~/moi-uroki --provider mock_say --out ~/lessons --export-zip
```

`--input` указывает на **папку** — Combine обработает все `*.txt` внутри неё по алфавиту,
последовательно (по одной теме за раз — так же, как обычная генерация одного файла). Ошибка
одной темы (битый файл, сбой сети, ошибка API) не прерывает обработку остальных: в конце
печатается сводка (сколько тем готово / сколько с ошибками), а код возврата процесса ненулевой,
если хотя бы одна тема не прошла — удобно для скриптов. Полный список опций (`--concurrency`,
`--voice-es`/`--voice-ru`, `--no-normalize`, `--no-id3` и др.) — см. `combine/README.md`.

## Часть 4: Экспорт на iPhone

### 4.1 Экран библиотеки

1. На главном окне Combine → вкладка **Библиотека**
2. Найдите нужный урок (например, «Рассказ о себе»)
3. Нажмите **[⋯ Ещё]** → **«Экспорт ZIP»**

**Результат:**
- Архив создаётся сразу, **без диалога сохранения и без всплывающего подтверждения** — файл
  `lesson-04-hablar-de-mi-mismo.zip` появляется в вашей папке уроков (по умолчанию `~/lessons`,
  настраивается в [Параметры] — см. §3.1). Если не уверены, что экспорт прошёл — откройте папку
  через **[⋯ Ещё] → «Открыть в Finder»**
- Размер: ~34 МБ
- Структура внутри:
  ```
  lesson.json (манифест)
  audio/
  ├── es/ (81 MP3)
  └── ru/ (81 MP3)
  ```

### 4.2 Три способа передачи на iPhone

#### Способ 1: AirDrop (самый быстрый)

1. На macOS: Finder → папка уроков (по умолчанию `~/lessons`, либо сразу через **[⋯ Ещё] →
   «Открыть в Finder»** в Библиотеке Combine) → `lesson-04.zip`
2. Right-click → AirDrop
3. Выберите свой iPhone
4. На iPhone: примет файл и откроет в Audio Learner

#### Способ 2: iCloud Drive

1. На macOS: Finder → iCloud Drive → создайте папку `AudioLearner`
2. Перетащите `lesson-04.zip` в эту папку
3. На iPhone: Files.app → iCloud Drive → AudioLearner → lesson-04.zip
4. Нажмите на файл → откроется в Audio Learner

#### Способ 3: Email / облако

1. На macOS: отправьте себе `lesson-04.zip` по email
2. На iPhone: откройте email → tap на вложение → "Copy to Audio Learner"

### 4.3 Альтернатива: экспорт в Anki (.apkg, новое в 1.1)

Если хотите повторять фразы в Anki — например, параллельно с Audio Learner, на любом устройстве,
не только на iPhone:

1. В Библиотеке → **[⋯ Ещё] → «Экспорт в Anki»**
2. `.apkg`-файл создаётся тем же образом, без диалога — `~/lessons/04-hablar-de-mi-mismo.apkg`
   (без приставки «lesson-», в отличие от ZIP)
3. Перенесите файл на устройство с приложением Anki (AirDrop/iCloud/email — как в §4.2) и
   откройте его в Anki — импортируется колода с названием урока

Колода = название урока, карточки собраны из всех фраз и слов; текст рассказа (`story`) в колоду

## Часть 6: Обучение на iOS

Четыре режима сессии: три аудио-режима (**Один раз**, **Цикл фраз**, **Цикл сессии**) и
**Флеш-карты** — интерактивный, без фонового аудио (новое в 1.1, см. §6.3). Плюс отдельная
точка входа — **Сессия дня**, кросс-урочная (новое в 1.2, см. §6.1).

### 6.1 Создание сессии

**Вариант А — сессия по одному уроку:**

1. **Уроки** → нажмите на урок → **[▶ Играть]**
2. **Выбор фраз:**
   - Фильтр по статусу: Все / Учу / В процессе / Выучено
   - Выберите нужные фразы (checkboxes)
   - Нажмите **[Далее →]**
3. **Параметры сессии** — см. §6.2 (для аудио-режимов) или §6.3 (для флеш-карт)
4. **[▶ Начать]**

**Вариант Б — «Сессия дня» (новое в 1.2):**

На вкладке **Сессия**, над списком уроков — плашка **«Сессия дня»** с подписью «Автоподбор фраз
к повтору по всем урокам». Она сама собирает фразы, которые пора повторить (по срочности и
давности последнего повтора), сразу из **всех** импортированных уроков — не нужно открывать
уроки по одному.

Лимит и порядок подбора настраиваются в Параметры → «Сессия дня»:
- **Лимит фраз** — по умолчанию 30
- **Порядок подбора** — «Сначала слабые» (сначала самые срочные, затем самые давно
  повторявшиеся — по умолчанию) или «Вперемешку»

Если повторять сейчас нечего — появится диалог «Всё повторено: сейчас нет фраз к повтору.
Возвращайтесь позже.» Прогресс каждой фразы по-прежнему обновляется в её родном уроке; сама
сессия дня не привязана к одному уроку и в истории отмечена как «Сессия дня». Запустить её
можно и без открытия приложения — с Home Screen виджета или через Siri (см. §6.5).

### 6.2 Параметры сессии и плеер (аудио-режимы)

Экран «Параметры сессии» для трёх аудио-режимов (у флеш-карт свой набор — см. §6.3):

| Параметр | Значения | По умолчанию |
|---|---|---|
| Режим сессии | Один раз / Цикл фраз / Цикл сессии / Флеш-карты | Один раз |
| Повторений | 1–10 | 5 |
| **Порядок сторон** (новое в 1.2) | ES→RU / RU→ES / ES→ES (shadowing) | ES→RU |
| **Пауза между сторонами** (новое в 1.2) | Фиксированная (0–15 сек) или Пропорциональная (длина стороны × коэффициент 1.0–2.5) | Пропорциональная, ×1.5 |
| **Автоскорость** (новое в 1.2, «Замедлять слабые фразы») | вкл/выкл — Учу ×0.75, В процессе ×0.9, Выучено ×1.0 | выкл |
| Скорость озвучки | 0.5x–2.0x (без изменения высоты тона) | 1.0x |
| Текст на lock screen | Оригинал + перевод / Только оригинал / Только перевод / Скрыть | Оригинал + перевод |
| Обновлять статусы фраз | вкл/выкл | вкл |

**Порядок сторон ES→ES** — режим для тренировки произношения (shadowing): проигрывает
испанский вариант, пауза, затем **снова** испанский (без перевода вообще) — удобно для
повторения вслед за диктором без опоры на русский текст.

**Пропорциональная пауза** — длительность паузы после стороны считается от длительности только
что сыгранного аудио (× коэффициент), а не фиксированным числом секунд: короткие фразы получают
короткую паузу, длинные — длинную, вместо одной и той же паузы для всех фраз урока.

**Плеер** во время сессии показывает текущую/следующую фразу, прогресс сессии, кнопки
⏮/⏸/▶/⏭, ❤️ Избранное, 🔁 Повторить фразу. Плюс (новое в 1.2):

- **🌙 Таймер сна** — выкл / 5 / 10 / 15 / 30 / 45 мин. По истечении сессия не обрывается
  резко: текущий повтор доигрывается до конца, затем плеер мягко ставит паузу (с
  хаптик-откликом) — сессию можно продолжить вручную в любой момент. Оставшееся время
  показывается рядом с прогрессом сессии в формате `M:SS`.

**На lock screen (экран блокировки)** — без изменений, Now Playing с текстом по вашей настройке
и стандартными кнопками управления воспроизведением.

### 6.3 Режим флеш-карт (новое в 1.1)

Четвёртый, интерактивный режим сессии — без фонового аудио и без lock screen: карточка на
экране, вы сами решаете, знали фразу или нет.

**Настройки в экране «Параметры сессии»** (вместо аудио-настроек из §6.2):
- **Направление** — Испанский → русский (по умолчанию, вопрос на испанском) или Русский →
  испанский
- **Автоозвучка карточки** — вкл по умолчанию: вопрос (и затем ответ) озвучиваются сами при
  показе, без ручного нажатия на 🔊

**Как это выглядит:**
1. Показывается **вопрос** — сторона по выбранному направлению, с кнопкой 🔊 для озвучки
2. **«Показать ответ»** → появляется вторая сторона (+ озвучка, если автоплей включён)
3. Вы отмечаете **«Знал»** ✅ или **«Не знал»** ❌ — кнопками либо свайпом карточки (вправо =
   «Знал», влево = «Не знал»)

**Что происходит при каждом ответе:**
- **«Знал»** — обычное продвижение SRS (после 3 повторов «Учу» → «В процессе», после 8 → «Выучено» — как и в аудио-режимах)
- **«Не знал»** — счётчик повторов растёт, но статус НЕ повышается, а сама карточка
  возвращается в конец колоды текущей сессии — она попадётся снова, пока вы не ответите
  «Знал». Итоговый счётчик закрытых карточек не задваивается при повторных показах — считаются
  только уникальные карты

**Итоги сессии** для флеш-карт дополнительно показывают метрику **«Точность (с первого раза)»**
— долю карточек, отвеченных «Знал» без единого «Не знал» до этого. У аудио-режимов такой строки
нет.

### 6.4 Завершение сессии

После того как все фразы прослушаны (или все карточки закрыты — для флеш-карт):

```
Сессия завершена! 🎉

Урок: Рассказ о себе
Дата: 21 июля 2026, 15:34

📊 РЕЗУЛЬТАТЫ:
• Фраз завершено: 35 из 35 ✓
• Время сеанса: 8 мин 45 сек
• Среднее время на фразу: 15 сек

📈 ОБНОВЛЕНИЯ СТАТУСА:
• 5 фраз: learning → inProgress
• 3 фразы: inProgress → mastered

✨ ДОСТИЖЕНИЯ:
• Первый шаг 🎯
• Три подряд ⏱ (3 сессии за 3 дня)

[Вернуться в уроки] [Новая сессия] [Статистика]
```

Для флеш-карт вместо «Фраз завершено» показывается «Карточек» и добавляется строка «Точность (с
первого раза): N%» (см. §6.3).

**Что происходит с фразами:**
- Новые фразы (learning): после 3 повторений → inProgress
- В процессе (inProgress): после 8 повторений → mastered
- Выученные (mastered): не меняются

### 6.5 Системная интеграция (новое в 1.2)

Три способа начать или отслеживать сессию без захода в приложение (iOS 17+):

**Home Screen виджет.** Средний размер (`systemMedium`) показывает сегодняшнюю статистику
(минуты, сессии, streak 🔥) и кнопку **«▶ Сессия дня»** — запускает сессию дня прямо с экрана
Домой, без открытия приложения. Маленький размер и lock-screen виджеты (circular/rectangular/
inline accessory) показывают только статистику, без кнопки запуска. Если фраз к повтору нет —
появится диалог «Всё повторено».

**Siri / Быстрые команды.** Работают фразы «Запусти испанский», «Сессия дня», «Начни сессию
дня» (плюс название приложения) — запускают ту же «Сессию дня», что и кнопка на виджете.

**Live Activity + Dynamic Island.** Пока идёт **аудио-сессия** (для флеш-карт Live Activity не
создаётся), на экране блокировки и в Dynamic Island показывается название урока, текущая фраза
(по вашей настройке «Текст на lock screen»), прогресс X/N и кнопка паузы. Обновляется только при
смене фразы — не каждую секунду — это экономит батарею, но означает небольшую задержку между
сменой фразы на экране плеера и на Dynamic Island.

### Общие вопросы

**Q: Можно ли менять голоса после генерации?**
A: Нет, выбор голоса фиксируется в lesson.json. Если хотите другой голос — переделайте генерацию (это добавит затраты).

**Q: Как обновить урок без потери прогресса?**
A: Используйте [Обновить] при импорте (а не [Заменить]). Это скопирует новые аудиофайлы, но прогресс останется.

**Q: Сколько времени занимает генерация 81 фразы?**
A: 15–20 минут (зависит от лимитов ElevenLabs). В фоне можно свернуть Combine — процесс продолжится.

**Q: Может ли быть ошибка парсера из-за переводов?**
A: Нет, парсер не смотрит на содержание. Проверяет только формат: наличие разделителя `|` (ровно одного на строку) и структуру блоков.

**Q: Что если оставить API-ключ пустым?**
A: Combine всегда работает в mock-режиме (бесплатно). Если позже захотите платный — введите ключ в параметры, он сохранится (см. §1.4).

**Q: Флеш-карты обновляют ту же статистику SRS, что и аудио-режимы?**
A: Да, это тот же механизм (статусы learning/inProgress/mastered, streak, календарь). Отличие только в том, что «Не знал» не продвигает статус и возвращает карточку в конец текущей колоды — до тех пор, пока вы не ответите «Знал».

**Q: Сессия дня — это отдельный урок в библиотеке?**
A: Нет, она просто собирает фразы к повтору из всех ваших уроков в одну сессию; прогресс каждой фразы по-прежнему обновляется в её родном уроке. В истории сессий такая сессия подписана как «Сессия дня».

### Обучение SRS (Spaced Repetition)

Каждая фраза проходит три состояния:

1. **learning** → 🆕 новая
   - Условие: никогда не повторяли или мало повторяли
   - Переход: после 3 повторений → inProgress

2. **inProgress** → 🔄 в процессе
   - Условие: повторяли, но не достаточно
   - Переход: после 8 повторений → mastered

3. **mastered** → ✅ выучено
   - Условие: хорошо выучена
   - Переход: нет, осталась выученной

Ответ «Не знал» на флеш-карте не продвигает состояние (см. §6.3) — считается только «Знал».

**Совет:** Сначала выбирайте только фразы со статусом «Учу» (learning) — это будут новые. После 3 сессий они перейдут в «В процессе».
