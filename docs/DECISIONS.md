# Архитектурные решения и отклонения от спецификации

**Статус:** живой документ. Здесь фиксируются все осознанные отклонения от файлов SPEC_*.md и принятые решения.
Спецификации в этой папке — исходное ТЗ; при противоречии **этот файл и `shared/lesson.schema.json` имеют приоритет**.

---

## D-01. Аутентификация ElevenLabs: `xi-api-key`, а не `Bearer`

DEPLOYMENT.md §1.3 указывает `Authorization: Bearer sk-...`. Реальный API ElevenLabs использует заголовок
`xi-api-key: <ключ>`. Реализуем по реальному контракту API (сверяться с https://elevenlabs.io/docs/api-reference).

## D-02. Голоса загружаются динамически через `GET /v1/voices`

Голоса из спеки («Pablo», «Masha» и т.д.) — вымышленные плейсхолдеры. Реальные voice_id — непрозрачные строки.
Combine получает список голосов запросом `GET /v1/voices`, фильтрует/помечает по языкам и даёт прослушать preview
(поле `preview_url` из ответа API). В `lesson.json` голос сохраняется как объект `{id, name}`.

## D-03. ID модели Flash: `eleven_flash_v2_5`

В спеке встречается `eleven_flash_v2.5` (с точкой). Реальный model_id — `eleven_flash_v2_5`. Список моделей можно
получать через `GET /v1/models`; в UI зашиты две основные + возможность указать произвольный ID.

## D-04. Mock-провайдер TTS на базе macOS `say` (ключевое улучшение)

Требование: полный локальный цикл разработки и проверки **без единого платного запроса**.
Combine имеет два провайдера:

- `elevenlabs` — реальный API (не вызывается в тестах вообще);
- `mock_say` — синтез через `say -v Mónica / -v Milena` → WAV (LEI16@22050) → MP3 (lamejs, чистый JS, без ffmpeg).

Mock-режим генерирует **настоящую слышимую речь**, поэтому пригоден для проверки всего конвейера, UX плеера на iOS
и черновой оценки уроков. Провайдер выбирается в настройках; без введённого API-ключа приложение работает в mock.

## D-05. Дешёвый тест-режим коротких фраз (требование пользователя)

На экране настроек — секция «Тестовая генерация»: короткий текст (по умолчанию ~25 символов), выбор голоса/модели,
кнопка «Сгенерировать тест» → один TTS-запрос → мгновенное прослушивание + фактическая стоимость.
Работает и в mock (бесплатно), и с реальным ключом (~$0.003 за фразу). Назначение: проверка ключа, качества голоса
и корректности кода перед дорогой генерацией целой темы.

## D-06. Цены не хардкодим

Тарифы ElevenLabs меняются; в спеке цифры ($0.10/1000) уже приблизительны. Цена за 1000 символов для каждой модели —
редактируемое поле в настройках (с дефолтами по документации). Калькулятор стоимости считает от этих значений и
всегда помечает результат как оценку.

## D-07. iOS deployment target: 17.0; проверка на симуляторе iOS 26.2

Спека противоречива (iOS 15+ но `@Observable` из iOS 17). Телефон пользователя — iOS 26.5.2.
Решение: target **iOS 17.0**, современный стек (`@Observable`, NavigationStack), сборка и тесты на симуляторе
iOS 26.2 (Xcode 26.2). CoreData оставляем (по спеке), SwiftData не используем.

## D-08. iOS-проект генерируется XcodeGen

`project.yml` — источник истины, `*.xcodeproj` в git не хранится (генерируется `xcodegen generate`).
Это позволяет агентам и CI собирать проект без ручных манипуляций в Xcode.

## D-09. Combine: Vite вместо webpack, electron-vite

Спека упоминает webpack/CRA (`react-scripts`) — устаревшее. Используем `electron-vite` (main + preload + renderer),
React 18 + TypeScript strict + Tailwind. Renderer умеет запускаться в обычном браузере с mock-IPC (`npm run dev:web`)
— для быстрой визуальной проверки UI без Electron.

## D-10. Headless CLI-режим Combine

`npm run cli -- generate --input <файл.txt> --provider mock_say --out <папка>` — генерация без UI.
Назначение: интеграционные тесты (Combine → ZIP → iOS), пакетная генерация 50 тем в будущем, отладка.

## D-11. lesson.json: schema_version и расширенный config

Добавлено поле `schema_version` (semver, сейчас `1.0`) — iOS проверяет совместимость при импорте.
`config` хранит `provider` и голоса объектами `{id, name}` (см. D-02). Полная схема: `shared/lesson.schema.json`.
JSON валидируется в тестах обоих приложений против этой схемы.

## D-12. Никаких реальных сетевых вызовов в тестах

Тесты Combine поднимают локальный HTTP-стаб (инжектируемый baseUrl) и проверяют retry/backoff/лимиты на нём.
Реальный ElevenLabs API в тестах не вызывается никогда (платные запросы). Реальную генерацию проверяем вручную
вместе с пользователем, когда появится ключ.

## D-13. Оценка длительности MP3 — по данным файла, а не по словам

Спека предлагала оценивать длительность по числу слов. Точнее и просто: считать из PCM при mock-генерации и
парсить длительность MP3 (music-metadata) при реальной. `duration_ms` в lesson.json — фактические миллисекунды.

## D-14. Исправления арифметики спеки

README спеки: «Общий размер ~1.7 ТБ» — на самом деле ~1.7 ГБ (34 МБ × 50 тем). На проектирование не влияет.

## D-15. Монорепо

```
espanol/
├── docs/        — спецификации и решения (этот файл)
├── shared/      — контракт: lesson.schema.json, образцы уроков, фикстуры
├── scripts/     — служебные скрипты (make-fixture.mjs)
├── combine/     — Desktop-генератор (Electron + React + TS)
└── ios/         — Audio Learner (SwiftUI, XcodeGen)
```

Ветки: `main` — стабильная; фичи агентов — в worktree-ветках, merge после прохождения тестов.

## D-16. iOS CoreData-модель — программная (NSManagedObjectModel в коде), не .xcdatamodeld

Модель CoreData на iOS строится программно в `CoreDataStack.makeModel()` (единый кэшированный
`NSManagedObjectModel` на процесс), а не в `.xcdatamodeld`. Причина: проект генерируется XcodeGen и
собирается headless через `xcodebuild` без Xcode-GUI (D-08); программная модель полностью
контролируема из кода, тривиально тестируется на in-memory сторе и не зависит от компиляции
`.xcdatamodeld`. Сущности, атрибуты и связи (с инверсиями и delete rules) описаны в коде.

## D-17. LearningSession переживает удаление урока (nullify)

Связь `Lesson → sessions` имеет delete rule **nullify** (а не cascade): при удалении урока его
`LearningSession` не стираются, а лишь отвязываются (`session.lesson = nil`). Иначе общая история
активности, streak и heatmap «худели» бы задним числом при удалении урока. Поэтому
`LearningSession.lesson` объявлен опциональным, а `StatisticsService` не обращается к `session.lesson`
(считает по `completedAt`/`actualDurationSeconds`), корректно работая с «осиротевшими» сессиями.
Миграция не нужна — продовых данных ещё нет.

## D-18. Lock-screen widget: только статистика (accessory), текущую фразу показывает Now Playing

Полноценный lock-screen виджет с текущей фразой (SPEC §7.1) не делаем: фразу на экране блокировки
уже показывает Now Playing (MPNowPlayingInfoCenter, D-… lock screen). Виджет реализован как единый
`StatisticsWidget` со статистикой дня и семействами Home Screen (`systemSmall`/`systemMedium`) и
lock-screen accessory (`accessoryCircular`/`accessoryRectangular`/`accessoryInline`). Так избегаем
дублирования и сложной синхронизации live-состояния сессии в виджет.
## D-19. GenerationQueue: одна задача = фраза целиком (ES+RU последовательно), не два элемента

`phrase.status` в схеме — одно поле на фразу, а не отдельно по языкам. Если бы очередь обрабатывала
ES и RU как два независимых элемента, пришлось бы придумывать способ свести их в единый статус
(что делать, если ES уже done, а RU ещё generating или failed?) — то есть держать состояние, которого
в контракте попросту нет. Решение: `GenerationTask` — одна фраза, внутри себя синтезирующая сперва
ES, затем RU последовательно; статус (`pending → generating → done|failed`) выставляется только этим
одним заданием, что естественно остаётся атомарным для схемы. `concurrency` (1–5, §4.2 спеки) —
число одновременно обрабатываемых ФРАЗ (то есть до `concurrency × 2` запросов в моменте на пике,
не считая задержки между ES/RU внутри задачи), а не количество HTTP-запросов напрямую — оговорка
для тех, кто будет тюнинговать под реальные рейт-лимиты ElevenLabs.

Идемпотентность (§4.3, §9.2 — "resume берёт только pending+failed") реализована по полю `status`
в lesson.json, а не по наличию файла на диске: при возобновлении обрабатываются только элементы,
у которых `status !== 'done'`. Это проще и предсказуемее, чем сверка с файловой системой, и совпадает
с буквальной формулировкой спеки ("кэш по статусу done").

## D-20. «Переделать всё» / «переделать only failed» переиспользуют config из lesson.json, не текущие настройки

Экран «Библиотека» (§4.4) регенерирует урок, для которого исходный текст мог уже не сохраниться
(lesson.json не хранит сырой .txt — только разобранную структуру). `startRegenerate()` поэтому берёт
provider/model/голоса/stability/similarity_boost/seed из уже сохранённого `config` этого урока, а не
из текущих настроек приложения — иначе частичная переделка (только failed) могла бы намешать в одном
уроке разные голоса/модели. Параметры очереди (concurrency/retries/delay/timeout), не влияющие на
содержимое аудио, по-прежнему берутся из текущих настроек.


## D-21. Флеш-карты: дизайн режима (v1.1)

Четвёртый режим сессии наряду с тремя аудио-режимами (`PlaybackMode.flashcards`), выбирается в
`SessionConfigView`. Интерактивный, без фонового воспроизведения/lock screen.

- **Карточка:** крупно текст вопроса + кнопка озвучки; автоозвучка — настройка сессии
  (`flashcardAutoplay`, по умолчанию вкл). Направление — настройка `flashcardDirection`:
  ES→RU (дефолт) или RU→ES.
- **Показать ответ** → вторая сторона + её аудио (автоплей, если включён).
- **«Знал»:** `reviewCount+1`, `PhraseStatistics.correctCount+1`, `lastReviewDate`, обычное
  SRS-повышение (3/8). **«Не знал»:** `reviewCount+1` без `correctCount`, state не повышается,
  карта возвращается в конец колоды текущей сессии (повтор до «Знал»). В счёт фраз сессии
  (`phrasesCompletedCount`) повтор не задваивает — считаются уникальные закрытые карты.
- Реализовано через `SpacedRepeatService.registerReview(_, wasCorrect:)`: при `false` растёт
  только `reviewCount`/`totalReviewCount`, без `correctCount` и без повышения state. Аудио-режимы
  вызывают с `wasCorrect: true` по умолчанию.
- **Итоги:** `SessionCompletedView` + новая метрика «Точность: N% с первого раза»
  (`SessionResult.accuracy`, считается в чистом `FlashcardSession`).
- **Аудио:** существующий `SessionPlayerService.playClip(_:speed:volume:)` — по одной фразе,
  без очереди повторов.
- **Свайпы** как альтернатива кнопкам: влево — «не знал», вправо — «знал».
